# The GTM Operating System: Week 1 Rhythm

You don't have a product problem. You have an execution problem.

Most founders can build something. Not many can build something AND create demand for it.

## The Daily Rhythm (Week 1)

**6:57 AM — Trigger Scan (60 min)**
Find 3-5 people showing visible buying signals. No fishing. No cold lists.

**8:03 AM — Outreach (90 min)**
Draft personalized messages. Position the offer, not the brand.

**9:30 AM — Build/Deliver (150 min)**
Week 1: Build the offer doc. Make sure you can articulate it in one sentence.

**12:00 PM — Follow-up (60 min)**
Reply to responses. Schedule calls. Update tracking.

**1:00 PM — Closing/Content (120 min)**
Batch create content for the week. Test positioning angles publicly.

**3:00 PM — Daily Log (30 min)**
Count: messages sent, responses received, calls booked, revenue closed.

## The Checkpoint

By end of Week 1:
- ✓ Offer doc exists (one-pager)
- ✓ 20 messages sent to real targets
- ✓ Response rate tells you if positioning works

If response rate <10%: Your offer isn't clear. Rewrite and test again.

If response rate >20%: You're onto something. Scale it.

## The Mindset

This is not about perfection. This is about learning.

Send the message. See what happens. Adjust.
